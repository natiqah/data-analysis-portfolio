#Week 10 - SQL + Excel Analysis Implementation

📌 Project Overview

	This project demonstrates data extraction using SQL and basic visualization using Excel. The focus is on transforming raw transactional data into structured outputs for exploratory analysis and reporting.

	Unlike dashboard-based BI tools, this project emphasizes manual analytical thinking and foundational reporting using Excel.

🎯 Objectives
	1. Perform data extraction and aggregation using SQL
	2. Clean and structure dataset for analysis
	3. Export query results into Excel
	4. Create basic charts for exploratory insights
	5. Develop foundational understanding of business metrics

🧾 Dataset used
	Superstore (from Kaggle)  

🛠 Tools Used
	SQL (data extraction and aggregation)
	Microsoft Excel (basic visualization)

🔄 Workflow
	Raw Dataset
	   ↓
	SQL Querying & Aggregation
	   ↓
	Export to Excel
	   ↓
	Basic Charts & Pivot Tables
	   ↓
	Exploratory Insights

📊 SQL Analysis Performed
	1. Monthly sales trend
	2. Profit by discount level (0.05 bucket)
	3. Profit margin by category
	4. Top 10 products by sales
	5. Average order value by segment
	6. Sales by region
	8. Shipping usage analysis

📈 Excel Output

	Excel was used for:

	Pivot tables for summarization
	Basic bar/line charts
	Exploratory visualization of SQL outputs

⚠️ This project focuses on static analysis rather than interactive dashboards.

💡 Key Insights
	- Sales show seasonal variations
	- Discount does not always improve profitability
	- A small number of products drive majority of revenue
	- Corporate segment shows higher average order value
	- Standard shipping is the most preferable despite has longest delivery days

🚀 Outcome

	This project demonstrates foundational skills in:

	SQL querying
	Data summarization
	Basic Excel reporting
	Early-stage business analysis
